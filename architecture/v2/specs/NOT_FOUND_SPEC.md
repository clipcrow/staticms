# NotFound 画面仕様書

## 概要

ユーザーが存在しない URL
にアクセスした場合や、アクセス権限のないリソースを表示しようとした場合に表示されるエラー画面です。
アプリケーションのどのルートにもマッチしない場合 (`*` route)
のフォールバックとして機能します。

## UI 構成

- **メインエリア (全画面中央揃え)**:
  - **エラーコード**: `404` (大きな見出し)。
  - **メッセージ**: `Page not found.` (簡潔な説明)。
  - **アクション**: `Go Home` ボタン。
    - クリックするとトップページ (`/`)、つまりリポジトリ選択画面へ遷移する。

## データフロー

- 特別なデータロードは発生しない。
- 認証状態 (`isAuthenticated`) に関わらず表示される可能性があるが、現状の
  `AppRoutes` では `RequireAuth`
  の外側に配置されていないため、認証後にのみ到達可能（未認証の場合は `/login`
  へ飛ばされる）。
  - **課題**:
    未認証ユーザーがデタラメなURLを叩いた場合も、一度ログイン画面へ飛ばされ、ログイン後に
    404 が出る挙動になる可能性がある。

## v1 からの課題と新規仕様案

- **リッチなナビゲーション**: 単に Home に戻すだけでなく、直前のページに戻る
  (`Back`) や、サイトマップ的なリンク集を表示する。
- **コンテキストに応じたメッセージ**:
  - リポジトリが見つからないのか (`Repo Not Found`)。
  - コレクション/記事が見つからないのか (`Content Not Found`)。
  - URL の誤りなのか。
  - 上記を判別してメッセージを出し分ける機能の検討。
- **ビジュアル**: イラストやアイコンを用いた親しみやすいデザインへの改善。
